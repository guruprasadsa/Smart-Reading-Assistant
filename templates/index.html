    ℹ️ About This Project

    🎯 Problem Solved

    Researchers and students waste hours manually reading through long PDFs and documents to find specific information. This system lets you upload any document and ask natural language questions, returning cited answers with source references and confidence indicators — so you know exactly how reliable each answer is.

    🏗️ Architecture

    [Upload PDF / DOCX / TXT] ↓ [Document Parser — PyMuPDF / python-docx] ↓ [RecursiveCharacterTextSplitter — 512 chars, 50 overlap] ↓ [Google embedding-2 → ChromaDB (persistent)] ↓ [MMR Retrieval (k=5, fetch_k=20, λ=0.7)] ↓ [Partial Answer Detection — confidence scoring] ↓ [Gemini 3.5 Flash (temp=0.2) → Answer + Citations] ↓ [Flask API → Web UI]

    🧠 Engineering Challenge: Partial Answer Detection

    Naive RAG systems either hallucinate when no chunk fully answers a question, or return irrelevant text with false confidence. Our system solves this by checking similarity scores before generation: if the best chunk scores below 0.75, the prompt switches to a hedged mode that honestly communicates uncertainty. Results are color-coded: ● HIGH, ◐ PARTIAL, ○ NOT FOUND.
